DAVID.COMPUTER

How This Site Works
Overview

DAVID.COMPUTER is a personal website built from scratch using fundamental web technologies. The site draws inspiration from two beautifully designed personal websites: liza.io and neil.computer.

Technology Stack

The website is intentionally built using only core web technologies, without any frameworks or build tools.

  • HTML5 for structure and semantic markup
  • CSS3 for styling and animations
  • Vanilla JavaScript for interactivity and theme switching
Key Features

The site includes several notable features:

  • Responsive design that adapts to different screen sizes
  • Dark/light theme toggle with persistent user preference
  • CSS variables for consistent theming
  • Custom typewriter animation effect
  • Semantic HTML for accessibility
  • Mobile-first approach to styling
Hosting and Domain

The website is hosted using GitHub Pages, which provides free hosting for static websites directly from a GitHub repository. The domain name "david.computer" was registered through Porkbun, a domain registrar known for competitive pricing and good customer service.

Version Control and Updates

All changes to the website are tracked through Git and hosted on GitHub. Visitors can view the progression of the site through the Changelog, which documents all updates, improvements, and fixes.

Directory Structure

The website follows a simple and organized structure:

  • index.html - Main landing page
  • styles.css - Global stylesheet
  • script.js - JavaScript functionality
  • /w/ - Content pages directory
  • /assets/ - Static assets (favicons, etc.)
  • /updates/ - Blog posts and updates
  • /projects/ - Project showcases
Security Considerations

The site implements several security best practices:

  • Content Security Policy headers
  • HTTPS enforcement
  • Secure external link handling
  • No sensitive data or server-side processing
  • Minimal JavaScript with no external dependencies
A Note on Transparency

Sharing these technical details is a common practice in the web development community. Since this is a static website with no backend or sensitive data, being open about its implementation poses no security risk. In fact, this transparency can help other developers learn and potentially contribute improvements.
